# v0.3 — Two Equal Top-Level Modes (PPT 演讲稿 vs 画图)

| | |
|---|---|
| 日期 | 2026-05-14 |
| 状态 | Spec — 用户跳过 plan 阶段直接进实现 |

## 0. 定位翻转

v0.1 / v0.2 的定位是"画图为核心，PPT 是包装"。v0.3 翻转为：**两个平等的顶级模式**。

```
/ai-draw <需求>
   ↓
[Step 1] 意图识别 → PPT 演讲稿 OR 画图
   ↓
   ├─ PPT 模式 ── full html-ppt 能力（36 themes / 31 layouts / 27 anim / 20 FX / 14 full-decks）
   └─ 画图模式 ── 现有 7 类型 × 8 themes（single + site）
```

## 1. 三个关键决策（已确认）

| 决策 | 选择 | 含义 |
|---|---|---|
| 主题系统 | 两套并存 | `assets/themes-diagram/` (8) + `assets/themes-ppt/` (36)，互不干扰 |
| 旧 deck 模式 | 退役 | v0.1 的 "deck" 走新 PPT 模式承袭；删除 `ppt/deck-template.html` |
| 混合模式 | 支持 | PPT 中可嵌画图（架构图、流程图等）作为一种 slide layout |

## 3. 模式路由

`SKILL.md` 重构为：

```
Step 0: 意图识别
   - 关键词或显式 flag → 锁定模式
   - 模糊 → 问"① 一份 PPT  ② 一张/几张图"

Step 1 (PPT 路径):
   - 沿 ppt/INSTRUCTIONS.md 的 3-问开场（内容 / 观众 / 全 deck 模板）
   - 推荐 3 主题（从 36 themes-ppt）
   - 推荐 1-2 个全 deck 模板（从 14 full-decks）
   - 生成多 slide HTML（含 mixed mode 时嵌入画图模板）

Step 1 (画图路径):
   - 沿现有 INTERACTION.md 流程
   - 推荐 3 主题（从 8 themes-diagram）
   - single / site
```

## 4. 主题分流规则

- **画图模式**：T 键仅在 8 themes-diagram 间循环；`Shift+T` 在全 8 间循环；`data-themes` attr 列举 3 推荐
- **PPT 模式**：T 键仅在 36 themes-ppt 间循环；推荐机制按 audience 选 3 个；`data-themes` attr 列举 3 推荐
- **混合 slide（PPT 中嵌画图）**：画图复用当前 PPT theme 的 `--bg` / `--text-1` / `--accent`，**不强求 `--sem-*` 完美**——尽力而为，必要时让用户调

## 5. 命令面变化

| 命令 | v0.2 | v0.3 |
|---|---|---|
| `/ai-draw <需求>` | 单图 / deck 路由 | PPT / 画图 路由 |
| `--mode single` | 单图 | 画图模式的单图 |
| `--mode deck` | 多图 → deck | **退役**（→ `--mode ppt`） |
| `--mode ppt` | 不存在 | **新**：PPT 演讲稿 |
| `--mode site` | 多页画图 | 不变（仍属画图模式） |
| `/ai-draw add` | 加 slide 到 deck | 加 slide 到 PPT |
| `/ai-draw add --to <site>` | 加子页到 site | 不变 |

## 6. 混合模式：PPT 中嵌画图

PPT 的 slide layout 增加一种 `<section class="slide layout-diagram" data-diagram-type="architecture">`：

- Claude 写 PPT 时遇到"这页放架构图"的需求
- 调画图模式的 `diagrams/architecture/INSTRUCTIONS.md` 生成 SVG
- SVG 内联到 slide 里
- 颜色尽量用 `var(--accent)` / `var(--bg-2)` / `var(--text-1)`（PPT theme 都有的 token），避免依赖 `--sem-frontend` 这种画图专属的 token（PPT theme 没有）

## 7. 退役 v0.1 deck 模式

- 删 `ppt/deck-template.html`
- 删 `ppt/INSTRUCTIONS.md` 旧内容（重写为 PPT 全功能）
- 删 `ppt/README-template.md` 旧内容（重写）
- `INTERACTION.md` 删除老 "deck flow" 章节，新增 "PPT flow"

## 8. YAGNI（v0.3 不做）

- 不为 36 themes-ppt 重写一遍兼容性矩阵——直接复用 html-ppt 的判断习惯
- 不做 `/ai-draw migrate` 把旧 v0.1 deck 输出升级到 PPT mode
- 暂不做 PPT mode 的 export-png 逐 slide 渲染（沿用 html-ppt 已有的 `scripts/render.sh`）
